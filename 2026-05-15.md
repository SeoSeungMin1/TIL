## 배운 내용

- 클래스
    - 생성자 : 객체가 생성될 때 자동으로 호출되어 초기화 수행하는 특수 메서드 → 클래스명과 동일, 리턴 타입 없고 void도 사용 x
    - 상속 vs Inner클래스
        
        → 상속 : is-a 관계 (~은 ~이다), Inner : has-a 관계(~이 ~을 가진다).
        
- 타입 : 변수나 값이 가질 수 있는 데이터의 종류와 형식을 지정하는 자바의 기본(설계의 일부)
    - 기본 타입 : 변수 선언 시 값을 직접 저장, 객체가 아니므로 메서드 호출x / ex) int
    - 참조 타입 : 값 자체가 아닌, 값이 저장된 메모리 주소를 저장하는 타입 / ex) Integer ← wrapper클래스
        
        → wrapper 클래스 : 기본타입인 int, double, char은  메서드 사용 못해서 기본 타입을 감싼 wrapper클래스인 Integer,Double,Character 사용
        
    - final : 더 이상 변경 못하게 막는 제한자
        
        → 사용 이유 : 안정성과 명확성
        
    - static : 객체 없이 클래스 이름만으로도 변수, 메서드 접근 가능
- String
    - 불변 객체인 문자열 자료형 → 변수는 바뀔 수 있지만 객체 자체는 수정되지 않는다.
    - 문자열 리터럴 : 큰따옴표” “로 직접 작성된 문자열 값
    - 문자열 비교
        
        → ==이 아니라 equals() 사용. / ==은 두 변수가 같은 객체를 참조하는지를 확인, equals()는 문자열의 내용이 같은지 확인
        
- 다형성
    - 여러 형태를 가지는 것 + 상위 타입을 기준으로 하위 객체를 다루고 동일한 메소드 호출이 실제 객체에 따라 다르게 동작하는 객체지향의 특성
    - 자바에서 주로 부모 클래스나 인터페이스 타입으로 자식 객체를 참조하는 방식으로 다형성 구현
        
        → 인터페이스 : 반드시 구현해야하는 기능 규칙
        
    - 오버로딩 : 메서드 이름은 같지만 매개변수의 형태나 개수를 다르게 정의
    - 오버라이딩 : 상속받은 메서드를 자식 클래스에서 다시 정의
    - 컴파일 타임 : 소스 코드를 자바 컴파일러가 바이트코드로 변환하는 시점
    - 런타임 : 컴파일된 프로그램이 실제로 실행되는 시점
    - 다형성 사용 이유 : 코드 수정 최소화하여 기능 유연하게 확장.
- 상속
    - 클래스 사이의 관계를 만드는 가장 기본적인 도구(is-a관계)
    - 자식 클래스가 부모 클래스의 필드와 메서드를 물려받아 재사용하거나 확장
        
        → 물려 받는다 : 부모 클래스에서 정의된 변수나 메소드를 자식 클래스가 사용할 수 있다
        
        → 재사용한다 : 같은 코드를 여러 클래스에서 반복해서 쓰지 않아도 된다.
        
        → 확장한다 : 부모에게 없는 필드나 메소드를 자식이 새로 추가 가능
        
    - this → 내 객체의 속성,메서드 접근 시
    - super → 부모 클래스 접근 시

## 헷갈린 점

- 상속 관계랑 Inner 클래스 차이점이 헷갈렸다.

## 느낀점

다형성에서 오버로딩과 오버라이딩에 대해 알고 있었지만 이번 수업을 통해 둘이 차이점을 정확하게 이해하게 됐다.
