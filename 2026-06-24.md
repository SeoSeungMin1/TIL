## 배운 내용

- 비동기 - 콜백 함수
    - 다른 함수에 인자로 전달되는 함수
    - 전달받은 함수 내부에서 특정 시점에 실행됨
    - 새로운 문법이 아니라 함수를 값처럼 다룰 수 있는 일급 함수의 활용
    - 비동기 결과 처리 할 때 자주 사용
    - 동기 콜백
        - 콜백이 즉시 실행되는 방식
        - 배열 메서드에서 자주 사용
        - map, filter, forEach, reduce의 콜백은 기본적으로 동기 실행
        
        ```jsx
        const names = ['startupcode', 'adapterz', 'tetricode'];
        
        names.map(function (name) {
            console.log(name);
        });
        ```
        
    - 비동기 콜백
        - 콜백이 나중에 실행되는 방식
        - 작업을 예약해두고 결과가 준비되면 실행
        - setTimeout, setInterval, 파일 읽기, 네트워크 요청 등에서 사용
        
        ```jsx
        setTimeout(function () {
            console.log("1초 뒤 실행");
        }, 1000);
        ```
        
    - 동기 콜백과 비동기의 차이
        - 동기 콜백은 함수를 인자로 넘기는 형태
        - 비동기는 실행 시점이 나중인 처리 방식
        - 콜백이라고 무조건 비동기는 아니다
    - 배열 메서드와 콜백 (map)
        - 배열의 각 요소를 처리할 함수를 전달
        - 대부분 동기적으로 실행
        - 나머지 메서드도 콜백 받는 구조 같고 결과만 다름
            
            → filter : 조건을 만족하는 요소만 모아 새 배열 반환
            
            → reduce : 배열을 하나의 값으로 축약
            
            → forEach : 각 요소를 순회하면서 실행하되 새 배열 반환 없음
            
    - 함수를 반환하는 함수
        - 함수가 다른 함수를 반환하는 방식
        - 특정 값을 기억한 함수를 만들 수 있음
        - 클로저와 연결되는 개념
    - 함수 컴포지션
        - 여러 함수를 조합해서 새로운 함수를 만드는 방식
        - 작은 함수를 조립해 복잡한 로직을 만들 수 있음
    - 콜백 사용 시 문제점
        - 비동기 콜백 안에서 발생한 에러는 바깥의 동기 try/catch로 잡기 어려움
    - Callback Hell
        - 비동기 콜백을 계속 중첩해서 사용하는 문제
        - 들여쓰기가 깊어짐
        - 가독성과 유지보수성이 떨어짐
        - 에러 처리 복잡해짐
        - 해결하기 위해 Promise 등장
        - .then()으로 흐름을 평평하게 작성 가능하다
        - .catch()로 에러 모아서 처리 가능
- 비동기 - Promise
    - 미래에 완료될 비동기 작업의 결과를 담는 객체
    - 성공 또는 실패 결과를 나중에 처리할 수 있게 해줌
    - 사용 이유
        - 콜백 지옥 해결
        - 에러 처리 단순화
        - 비동기 작업 체이닝 가능
        - 여러 비동기 작업을 함께 관리 가능
    - Promise 상태
        - pending → 결과를 기다리는 상태
        - fulfilled → 성공 상태, resolve() 호출
        - rejected → 실패 상태, reject() 호출
        - fulfilled ↔ rejected 변경 불가능
    - 사용 방법
        - Promise 생성
        
        ```jsx
        const promise = new Promise((resolve, reject) => {
            resolve("성공");
        });
        ```
        
        - 결과 처리
        
        ```jsx
        promise
            .then(result => {})
            .catch(error => {})
            .finally(() => {});
        ```
        
        → then : 성공 시 실행, catch : 실패 시 실행, finally : 성공, 실패 관계없이 항상 실행
        
    - 동작 방식
        - Promise 생성 → pending 상태 → 비동기 작업 수행 → resolve 또는 reject 호출 → 상태 확정 → then 또는 catch 실행
    - 생성 방식
        - 선언 방식
        
        ```jsx
        const fetchNumber = new Promise((resolve) => {
            setTimeout(() => resolve(10), 1000);
        });
        ```
        
        → Promise 생성 시 즉시 실행, 한 번만 사용하는 작업에 적합
        
        - 함수 방식
        
        ```jsx
        function fetchNumber() {
            return new Promise((resolve) => {
                setTimeout(() => resolve(10), 1000);
            });
        }
        ```
        
        → 호출할 때마다 새로운 Promise 생성, 실무에서 더 자주 사용한다.
        
    - Promise Chain
        - 여러 비동기 작업을 순서대로 연결하는 방식
        - then의 return값이 다음 then으로 전달
    - Promise와 이벤트 루프
    
    ```jsx
    console.log(1);
    
    Promise.resolve()
        .then(() => console.log(3));
    
    console.log(2);
    
    setTimeout(() => console.log(4), 0);
    ```
    
    → 동기 코드 먼저 실행 → Promise.then() (마이크로태스크) → setTimeout() (매크로태스크)
    
    - Promise.all()
        - 여러 Promise를 동시에 시작
        - 모두 성공해야 then 실행
        - Promise.allSettled() → 성공, 실패 여부 상관없이 각각의 결과를 반환
    - 주의할 점
        - execuutor는 비동기가 아니다.
        
        ```jsx
        console.log("A");
        
        new Promise(() => {
            console.log("B");
        });
        
        console.log("C");
        // 출력 A -> B -> C
        ```
        
        → execuotr : new Promise() 안에 넘기는 함수 전체
        
- 비동기 - Async / Await
    - Promise를 더 쉽게 사용하기 위한 문법
    - 새로운 비동기 기술이 아니라 Promise 위에 만들어진 문법적 설탕
    - 비동기 코드를 동기 코드처럼 작성할 수 있게 해줌
    - async
        - 함수 앞에 붙이는 키워드
        - 해당 함수는 항상 Promise 반환
        - 일반 값을 반환해도 자동으로 Promise로 감싸짐
    - await
        - Promise 앞에 붙이는 키워드
        - Promise가 완료될 때까지 기다렸다가 결과 값 꺼냄
        - async 함수 안에서만 사용 가능
    - 사용 이유
        - Promise 체인(.then) 가독성 개선
        - 비동기 코드를 위에서 아래로 읽을 수 있다.
        - try/catch 사용 가능
        - 에러 처리 단순화
    - 사용 방법
        
        ```jsx
        async function 함수이름() {
            const 결과 = await 프로미스;
        }
        ```
        
        - 화살표 함수
        
        ```jsx
        const 함수이름 = async () => {
            const 결과 = await 프로미스;
        };
        ```
        
    - await의 진짜 동작
        
        ```jsx
        console.log('1');
        
        async function f() {
            console.log('2');
        
            await null;
        
            console.log('4');
        }
        
        f();
        
        console.log('3');
        // 출력 결과 1 2 3 4
        ```
        
        → async 함수는 await 만나기 전까지 동기 실행
        
    - 예외 처리
        - async/await에서는 비동기 작업을 `try/catch`로 처리 가능
        - Promise의 `.catch()`를 사용하는 것과 같은 역할
        - Promise가 실패(rejected)하면 await가 예외를 발생시켜 catch로 전달
    - Promise와 관계
        - async는 Promise를 자동 생성
        - await는 Promise 결과를 꺼내는 문법
        - 내부적으로는 여전히 Promise 사용
        - Promise 없이 동작하는 것이 아님
