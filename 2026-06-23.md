## 배운 내용

- 함수형 패러다임 - 익명함수
    - 이름을 붙이지 않은 함수
    - 새로운 문법이 아니라 함수 이름을 생략한 함수
    - 주로 함수 표현식이나 콜백 함수에서 사용
    - 사용 이유
        - 코드가 간결해진다. → 한 번만 사용할 함수를 따로 만들 필요 없다.
        - 임시 함수 사용 → 함수 이름을 관리할 필요 없음
        - 클로저 생성 → 외부 변수를 기억하는 함수 생성 가능
        - 이벤트 처리 및 콜백 → setTimeout, addEventListener, map 등에서 사용
    - 사용 방법
        - 함수 표현식
        
        ```jsx
        const greet = function () {
            console.log("Hello");
        };
        ```
        
        - 다른 함수의 인자로 전달
        
        ```jsx
        executeFunction(function () {
            console.log("Hello");
        });
        ```
        
        - 배열 메서드 콜백
        
        ```jsx
        users.map(function (user) {
            return user.name;
        });
        ```
        
        - 화살표 함수
        
        ```jsx
        users.map((user) => user.name);
        ```
        
    - 특징
        - 이름이 없음
        - 함수 객체는 정상적으로 생성됨
        - 변수 저장, 전달, 반환 가능
        - 실행할 때마다 새로운 함수 객체 생성
    - 함수 선언문 vs 익명 함수
        - 함수 선언문 : 호이스팅 가능, 선언 전 호출 가능
        - 익명 함수 : 호이스팅 불가능, TDZ때문에 선언 전 호출 불가능
    - 주의할 점
        - 이벤트 리스너 제거 문제 → 동일한 함수 참조가 아니면 제거 불가능
        - React 리렌더링 문제 → 리렌더링마다 새 함수 생성, 불필요한 리렌더링 발생 가능
- 함수형 패러다임 - 즉시 실행 함수
    - 정의되자마자 바로 실행되는 함수
    - 보통 익명 함수 형태로 작성
    - 함수를 만들고 바로 실행하기 때문에 이름이 필요 없음
    - 한 번 실행되고 끝나는 코드에 사용
    - 사용 이유
        - 변수와 함수의 스코프 제한
            
            → 함수 내부 변수는 외부에서 접근 불가능
            
            → 전역 변수 오염 방지
            
            → 변수 이름 충돌 방지
            
        - 즉시 실행
            
            → 페이지 로드 시 한 번만 실행할 초기화 코드에 적합
            
            → 설정값 계산, 이벤트 등록에 사용
            
        - 코드 캡슐화
            
            → 관련 코드를 하나의 영역으로 묶어 관리 가능
            
            → 클로저와 함께 사용하면 모듈처럼 활용 가능
            
    - 동작 방식
        1. 익명 함수 생성
        2. 괄호로 함수 표현식으로 만든다.
        3. 뒤에 ()를 붙여 즉시 실행
        4. 실행 후 종료
    - 사용 방법
    
    ```jsx
    (function() {
        console.log("실행");
    })();
    ```
    
    - 특징
        - 한 번 실행되고 종료됨
        - 독립적인 스코프 생성
        - 내부 변수는 외부에서 접근 불가능
        - 보통 익명 함수로 작성
        - 클로저와 함께 사용 가능
    - 장점
        - 전역 변수 오염 방지
        - 변수 이름 충돌 방지
        - 초기화 코드를 깔끔하게 작성 가능
        - 코드 캡슐화 가능
    - 과거에는 많이 사용 → var만 존재해서 변수 격리가 어려웠다.
    - 현재는 사용 빈도 감소 → let, const의 블록 스코프 사용, ESM 사용
- 모듈 시스템
    - 코드를 기능별 파일로 분리해서 관리하는 방식
    - 자바스크립트에서 모듈은 파일 하나를 의미
    - 사용 이유
        - 코드 분리 → 기능별로 파일을 나누어 관리 가능
        - 재사용성 향상 → 필요한 기능만 다른 파일에서 가져와서 사용 가능
        - 유지보수 용이 → 관련 코드끼리 모여 있어서 수정 쉽다.
        - 캡슐화 → 외부에 공개할 것만 export 가능, 내부 구현 숨김 가능
    - CommonJS(CJS)
        - Node.js에서 사용하던 전통적인 모듈 방식
        - `module.exports` 로 내보냄
        - `require()` 로 가져옴
        
        ```jsx
        // module.exports 사용 방법
        module.exports = { add, subtract };
        // require() 사용 방법
        const math = require('./math.js');
        ```
        
        - 특징
            
            → 동기적 방식 : 파일을 다 읽을 때까지 다음 코드 실행 안 함
            
            → 동적 방식 : require를 조건문이나 함수 안에서 사용 가능
            
            → 값 복사, 가져오는 순간의 값 사용
            
    - ECMAScript Module(ESM)
        - 자바스크립트 공식 표준 모델 시스템
        - React, Vue 등에서 사용
        - export, import 사용
        
        ```jsx
        // export 사용 방법
        export function add(a, b) {
            return a + b;
        }
        // import 사용 방법
        import { add } from './math.js';
        ```
        
        - 특징
            
            → 정적 방식 : import는 최상단에서만 사용 가능, 실행 전에 의존성 분석 가능
            
            → 비동기 방식 : 모듈 로딩을 효율적으로 처리
            
            → 라이브 바인딩 : 원본 값과 실시간 연결, 원본 값 변경하면 가져온 쪽에서도 변경된 값 확인 가능
            
        - Named Export
            
            → 여러 값을 이름으로 내보내는 방식
            
            ```jsx
            export function add() {}
            export function subtract() {}
            
            import { add, subtract } from './math.js';
            ```
            
            → import 시 이름이 동일해야 함
            
        - Default Export
            
            → 모듈 대표 값 하나를 내보내는 방식, 파일당 하나만 가능
            
            ```jsx
            export default {
              add(a, b) { return a + b; },
              subtract(a, b) { return a - b; }
            };
            
            import math from './math.js';
            console.log(math.add(10, 5));
            ```
            
            → import 시 이름 자유롭게 지정 가능
            
    - Tree Shaking
        - 사용하지 않는 코드를 제거하는 최적화 기법
        - ESM에서 효과적으로 동작
    - ESM에서 Tree Shaking 가능한 이유
        - import가 정적이기 때문
        - 실행 전에 어떤 코드를 사용하는지 분석 가능
        - 번들러가 사용하지 않는 코드를 제거 가능
- 블로킹, 논블로킹 / 동기, 비동기
    - 블로킹
        - 호출한 쪽이 작업이 끝날 때까지 기다리는 방식
        - 제어권을 바로 돌려받지 못함
        - 다른 작업을 수행할 수 없음
    - 논블로킹
        - 작업을 요청한 뒤 바로 제어권을 돌려받는 방식
        - 작업 완료를 기다리지 않음
        - 다른 작업을 계속 수행 가능
    - 동기
        - 결과를 받을 때까지 기다리는 방식
        - 코드 실행 순서와 결과 처리 순서가 같다
    - 비동기
        - 결과를 기다리지 않고 다음 작업 수행
        - 결과는 나중에 콜백, Promise 등으로 받는다
    - 블로킹 vs 논블로킹
        
        → ” 제어권을 바로 돌려주는가?”
        
        | 구분 | 블로킹 | 논블로킹 |
        | --- | --- | --- |
        | 제어권 | 안 돌려줌 | 바로 돌려줌 |
        | 호출한 쪽 | 멈춤 | 계속 실행 |
        | 기준 | 차단 여부 | 차단 여부 |
    - 동기 vs 비동기
        
        → “결과를 기다리는가?”
        
        | 구분 | 동기 | 비동기 |
        | --- | --- | --- |
        | 결과 처리 | 기다림 | 기다리지 않음 |
        | 실행 순서 | 순차적 | 독립적 |
        | 기준 | 결과 처리 | 결과 처리 |
    - JS에서 주로 사용하는 조합
        - 동기 + 블로킹 : 파일을 다 읽을 때까지 멈추고 읽기가 끝난 뒤 다음 작업 수행
        - 비동기 + 논블로킹 : 파일 읽기를 맡기고 바로 다음 작업 수행, 결과는 나중에 콜백으로 처리
    - 사용 이유
        - 성능 향상 → 여러 작업을 동시에 처리 가능
        - 사용자 경험 개선 → 화면 멈춤 현상 방지
        - 네트워크 요청 최적화
    - JS 비동기 처리 구조
        - 콜 스택
            - 현재 실행 중인 함수가 쌓이는 공간
            - 자바스크립트는 콜 스택이 하나뿐
            - 한 번에 하나의 작업만 실행 가능
        - Web API
            - 브라우저가 제공하는 기능
            - setTimeout, fetch, 이벤트 처리 등을 담당
            - 오래 걸리는 작업을 대신 수행
        - 태스크 큐 (Task Queue)
            - 완료된 비동기 작업의 콜백이 대기하는 공간
        - 이벤트 루프 (Event Loop)
            - 콜 스택과 태스크 큐를 계속 확인
            - 콜 스택이 비면 태스크 큐의 작업을 가져옴
    - 테스크 큐 종류
        - 매크로 태스크 큐 : setTimeout, setInterval, 이벤트 핸들러
        - 마이크로 태스크 큐 : Promise.then()
