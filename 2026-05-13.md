## 배운 내용

- JDK : 자바로 개발을 하기 위한 도구 모음
    - 컴파일러, 실행도구, 문서화도구, 패키징 도구 등
    - 키워드 : 개발, 실행, 도구 모음
    - 사용 이유 : 개발에 필요한 모든 도구를 통합해 빠르고 안정적으로 개발 가능
- JRE : 자바를 실행하기 위한 환경
- JVM : 자바 바이트 코드를 실행하는 가상 머신
    - 자바 소스 코드 → .class 바이트 코드로 컴파일 된 이후 실행
    - 소스 코드 : 사람이 읽기 좋은 코드 (main, system.out.println 등)]
    - javac 컴파일러가 소스 코드를 .class 파일로 컴파일 → JVM이 .class 파일 읽고 실행
    - 운영체제 상관 없이 동일하게 실행되는 실행환경 제공 → 실행 독립성
- JAVA 특징
    - 컴파일 언어와 인터프리터 언어 특징 모두 가짐
    - 소스 코드를 컴파일하여 바이트코드 생성 후 JVM 실행

## 헷갈린 점

- 바이트 코드와 네이티브 코드 차이
- JVM을 사용하는 이유

## 느낀 점

JDK , JRE , JVM 개념과 서로의 관계를 이해할 수 있었다.
